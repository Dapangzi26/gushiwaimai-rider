# 骑手端自动掉线调试记录

## 现象
- 用户未主动点击退出，骑手端自己掉回登录态或跳转登录页。

## 已知信息
- 现象发生在骑手端 App 使用过程中。
- 当前还未确认是页面鉴权重定向、token 被清空、接口 401、socket 断连误判，还是 App 进程重启。

## 待验证假设
- 假设 1：请求拦截器在收到 401 或特定错误码后，自动清除了登录信息并跳转登录页。
- 假设 2：某个页面的 `onShow` / `onLoad` 鉴权逻辑误判当前用户不是骑手，主动执行了 `reLaunch` 到登录页。
- 假设 3：本地存储里的 `token` 或用户信息被定时逻辑、异常处理或覆盖写入清空了。
- 假设 4：`socket` 断连或登录态校验失败后，被业务代码当成“掉线”处理并触发重新登录。
- 假设 5：App 发生闪退或原生页返回异常，重启后因为本地登录态未恢复，看起来像“自己掉线”。

## 下一步
- 检查登录态存储读写点、请求拦截器、鉴权守卫、socket 生命周期与跳转登录逻辑。

## 证据更新
- 本轮用户复现时，未产生 `.dbg/trae-debug-log-rider-auto-logout.ndjson`，说明没有走到已插入的 `401 / clearInvalidLocalSession` 上报分支。
- 用户截图出现系统弹窗：`HTML5+ Runtime / 打包时未添加push模块`。
- 代码中 `utils/reminder-center.js` 存在 `plus.push.createMessage` 与 `plus.push.addEventListener` 调用。
- `manifest.json` 当前 `app-plus.modules` 只声明了 `Speech`，未声明 `Push`。

## 假设验证
- 假设 1：请求拦截器收到 401 自动清会话。
  - 状态：暂未确认，本轮无对应调试日志，且现象不匹配“登录失效提示”。
- 假设 2：页面鉴权误判主动跳登录。
  - 状态：暂未确认，本轮截图未显示登录页。
- 假设 3：本地 token / userInfo 被清空。
  - 状态：暂未确认，本轮缺少清会话调试证据。
- 假设 4：socket 断连导致掉线。
  - 状态：基本排除，App 端 socket 已禁用。
- 假设 5：并非真正掉线，而是提醒中心调用 `plus.push` 时因未打包 Push 模块触发原生运行时弹窗/重启感知。
  - 状态：当前最强疑点，已有截图与代码静态证据支持。
