# [OPEN] 站长端超时而骑手端正常

## 症状
- 同时打开两个骑手端时，乡镇站长端频繁提示“网络错误，请检查网络”
- 乡镇普通骑手端可正常进入并加载
- 站长端控制台集中出现 `request:fail timeout`

## 当前证据
- `utils(工具)/request.js(请求封装)` 的失败分支把所有超时统一提示为“网络错误”
- `App.vue(应用入口)` 中，站长会启动后台任务与位置上报；普通乡镇骑手当前不会
- `pages(页面)/index(首页)/index.vue(首页)` 中，站长首页比普通骑手额外加载：
  - 站长消息未读
  - 商家审核待办
  - 骑手审核待办
- 这些额外请求任意超时，站长端都会出现“网络错误”

## 可证伪假设
1. 站长和骑手请求格式相同，但站长会多发 3-5 个接口，导致更容易触发超时。
2. 站长端的某个专属接口响应过慢，拖累首页加载并反复弹“网络错误”。
3. 站长端位置上报接口在弱网或并发场景下频繁超时，和首页请求叠加后放大问题。
4. 不是前端请求体差异，而是站长身份命中不同接口分支或不同后端查询复杂度。

## 下一步
- 逐一核对站长端比骑手端多出的请求
- 确认是首页统计类接口超时，还是位置上报超时
- 再决定是否需要降并发、静默后台请求或拆分站长首页加载节奏
