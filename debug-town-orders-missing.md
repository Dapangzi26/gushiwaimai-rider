# 乡镇订单消失调试记录

## 症状
- 乡镇骑手重启并同步后，看不到之前提到的 4 个订单。
- 站长控制台有报错，骑手控制台看起来正常。
- 当前截图里站长端日志主要是定位相关 fallback / timeout 日志。

## 预期
- 乡镇订单在新逻辑下应稳定进入待接单池。
- 乡镇骑手和站长端对同一批订单的可见性应符合新规则，不应出现“重启后订单消失”。

## 当前证据
- 站长控制台显示会话校验通过，后台任务已启动。
- 站长控制台日志主要集中在定位上报 fallback 与 timeout。
- 尚未拿到订单接口返回体和数据库中这 4 单的当前状态字段。

## 初始假设
1. 这 4 单仍是旧流转数据，状态或责任人字段不符合新待接单池规则，因此刷新后被前端正确过滤掉。
2. 站长端报错与订单无关，只是定位超时日志；真正导致“订单消失”的是接口返回数据已变化。
3. 后端重启后某段乡镇订单流转逻辑把旧订单责任人或状态改写了，导致新前端口径下两端都不显示。
4. 前端订单页与工作台首页虽然已统一口径，但站长端 / 骑手端对同一订单的筛选条件仍存在一处未对齐。
5. 当前看到的“4单”来自旧缓存或旧轮询快照，重启后快照清空，真实接口其实一直没返回这 4 单。

## 下一步证据点
- 抓 `外卖订单(订单列表)` 接口返回的原始订单数组。
- 核对这 4 单在数据库中的 `status / rider_id / current_responsible_user_id / current_responsible_role / dispatch_center_status`。
- 对照站长端与乡镇骑手端当前账号身份字段：`delivery_scope / rider_kind / town_name / town_code`。
